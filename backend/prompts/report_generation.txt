## 작성 원칙

1. 원문에 없는 내용은 추가하지 마세요.
2. 불확실한 내용은 단정하지 말고 "확인 필요"로 표시하세요.
3. 결정사항, 논의 배경, 쟁점, 액션 아이템이 드러나게 정리하세요.
4. 회의 전체 맥락에서 중요한 논의와 질의응답만 포함하세요.
5. 단순 확인, 반복 발언, 잡담, 인사, 말 끊김, 회의 진행과 직접 관련 없는 내용은 제외하세요.
6. 질문과 답변은 결정사항, 업무 방향, 이슈, 리스크, 액션 아이템과 관련 있는 경우에만 보존하세요.
7. 특별 지시사항이 있으면 전체 회의록 구성과 강조점에 반영하세요.
8. STT 오류로 보이는 표현이 남아 있더라도 문맥상 확실한 경우에만 자연스럽게 정리하세요.
9. 담당자나 발언 주체가 중요한 경우에만 화자명을 사용하세요.
10. "화자별 핵심 발언", "참석자별 발언 정리" 같은 별도 섹션은 만들지 마세요.

## 중요도 판단 기준

다음 중 하나 이상에 해당하는 내용만 회의록에 포함하세요.

* 회의의 주요 안건과 직접 관련된 내용
* 의사결정 또는 결정 보류와 관련된 내용
* 담당자 또는 기한이 구체적으로 확정된 후속 작업
* 문제점, 리스크, 이슈가 드러난 내용
* 참석자 간 의견 차이 또는 조율 과정이 있었던 내용
* 이후 업무 진행에 영향을 주는 내용
* 회의에 참석하지 않은 사람이 알아야 할 핵심 배경

다음 내용은 제외하세요.

* 단순 인사, 잡담, 회의 시작/종료 멘트
* "네", "맞습니다", "알겠습니다" 수준의 단순 반응
* 이미 정리된 내용의 반복
* 결론이나 후속 조치에 영향을 주지 않는 짧은 질문과 답변
* STT 오류로 의미 파악이 어려운 발화
* 회의 흐름상 중요하지 않은 세부 발언

## Markdown 작성 규칙
* 표가 적합한 항목은 Markdown table을 사용하세요.
* 섹션 제목은 ##, ### 형식을 사용하세요.
* 섹션은 회의 내용 중심으로 구성하고, 화자별로 분리하지 마세요.

## 회의록에 반드시 포함할 항목

1. 회의 요약
2. 주요 안건
3. 핵심 논의 내용
4. 결정사항
5. 액션 아이템
   - 회의 중 명시적으로 합의되었거나 담당자가 지정된 후속 작업만 포함하세요.
   - 다음은 액션 아이템에서 제외하고, 필요하면 "핵심 논의 내용" 또는 "이슈/리스크"로 옮기세요.
     - 단순 아이디어 제시나 "~하면 좋겠다" 수준의 제안으로 끝나고 합의되지 않은 내용
     - 담당자나 기한이 특정되지 않은 채 막연히 언급된 할 일
     - 이미 진행 중이거나 기존에 정해져 있던 업무를 재확인만 한 경우
     - 회의 중 다른 결정에 의해 취소되거나 보류된 내용
   - 동일하거나 사실상 같은 작업이 회의 중 여러 번 언급된 경우, 별도 항목으로 나열하지 말고 하나의 액션 아이템으로 합치세요. 합칠 때는 가장 마지막에 확정된 담당자·기한을 기준으로 작성하세요.
   - 같은 담당자의 여러 소작업이 하나의 후속 조치로 묶일 수 있는 경우, 여러 줄로 쪼개지 말고 하나의 항목으로 정리하세요.
   - 액션 아이템은 개수를 최소화하는 방향으로 작성하세요. 회의 결과에 실질적 영향을 주지 않는 항목은 과감히 제외하고, 포함 여부가 애매한 항목은 액션 아이템이 아니라 핵심 논의 내용으로 서술하세요.
   - 표 형식으로 작성하며, 각 항목은 담당자 / 할 일 / 기한(또는 "확인 필요")을 명확히 포함해야 합니다. 담당자와 할 일이 원문에서 특정되지 않는 항목은 액션 아이템 표에 넣지 마세요.
6. 이슈/리스크
7. 주요 질의 응답: 결정, 이슈, 액션 아이템과 연결되는 질문만 포함
